## YAML Frontmatter 规范

### 必填字段
```yaml
---
title: "文章标题"
date: 2026-02-28T11:31:00+08:00
description: "简短描述"
tags:
    - 标签1
    - 标签2
categories:
    - 分类1
---
```

### 可选字段
- `draft: false` - 草稿状态（不发布）
- `pinned: false` - 是否置顶
- `image: ""` - 封面图片路径
- `math: false` - 是否需要数学公式渲染
- `license: ""` - 协议证书
- `comments: true` - 是否允许评论
- `hidden: false` - 是否隐藏文章

### 注意事项
- ✅ 日期格式必须是 ISO 8601：`2026-02-28T11:31:00+08:00`
- ✅ 标签和分类用列表格式（缩进 4 个空格）
- ❌ 避免在 `description` 中使用冒号，容易破坏 YAML 解析
- ✅ 字符串用双引号包裹，避免特殊字符问题

## 标题与结构

### 标题层级规范
```markdown
# 一级标题（文章主题，仅一个）

## 二级标题（主要段落）

### 三级标题（子段落）
```

### 规则
- 一篇文章只能有**一个一级标题**（对应 YAML 中的 `title`）
- 二级标题用于主要内容分段
- 三级标题用于小节（通常在代码块前）
- 避免使用四级或更深的标题
- 标题前后各留**一行空白**

### 推荐的文章结构
```markdown
# 文章主标题

简短介绍段落（2-3 句话）。

## 背景/问题描述

展开说明问题背景。

## 核心内容

### 解法一

代码块...

### 解法二

代码块...

## 总结/性能对比

对比分析。

## 相关资源

链接列表。
```

## 算法题目分析格式

对于算法题目和编程问题的分析文章，应使用以下**四个板块**来组织内容。

### 板块结构

#### 1. 问题（## 问题）

清楚地阐述问题的核心要求，包括：
- 问题的背景或来源（如竞赛信息、平台名称）
- 输入和输出要求的说明
- 关键的问题陈述
- 必要的样例（输入/输出示例）
- 数据范围和约束条件

**示例：**
```markdown
## 问题

### 题目来源
NOIP 2000 提高组 T4

### 问题描述
设有 N×N 的方格图（N≤9），部分方格中填入正整数，其他为 0。
某人从左上角（0,0）出发，可向右或向下走，到达右下角（N,N）。
在路上可取走方格中的数（取后变为 0）。
从 A 到 B 走两次，找两条路径使得取数之和最大。

### 输入输出
- 输入：N（方格大小），然后是若干行"x y value"
- 输出：最大取数和

### 约束条件
- 1≤N≤9
- 数值为正整数
```

#### 2. 思路分析（## 思路分析）

分析问题的解决思路，包括：
- 问题分析和抽象
- 关键洞察和观察
- 子问题的定义
- 状态转移或算法逻辑的说明
- 时间和空间复杂度的初步分析

可用多个三级标题分别介绍不同的解法。

**示例：**
```markdown
## 思路分析

### 核心思想
采用动态规划，同步推进两条路径，用一个 DP 同时模拟两个人的行走。

### 状态设计
- `dp[k][x1][x2]`：k 为当前步数，x1、x2 分别为两人的行号
- y 值通过 `y = k - x` 推导，降低维度

### 关键处理
当两人在同一格时（x1 == x2），该格只计数一次

### 转移方程与逻辑
每一步两人各可向右或向下，共 4 种组合...
```

#### 3. 代码实现（## 代码实现）

完整的、可运行的代码，包括：
- 必要的头文件和全局变量声明
- main() 函数（如适用）
- 核心算法函数
- 保留解释性注释和美化分隔符
- 单个完整代码块可 120 行以内；超过则分多个块展示

**示例：**
```markdown
## 代码实现

```c
#include <stdio.h>

int N;
int map[10][10];
int dp[20][10][10]; // dp[步数][人1的行][人2的行]

int main() {
    // ... 实现
}
```

### 关键函数说明（如需）
可另行说明特定函数的逻辑或边界条件。
```

#### 4. 复杂度与优缺点（## 复杂度与优缺点）

对解法的性能和特点进行评估，包括：
- 时间复杂度和空间复杂度分析
- 解法的优点
- 解法的缺点或局限
- 与其他解法的对比（如有多个解法）
- 适用场景

**示例：**
```markdown
## 时间复杂度与优缺点

### 时间复杂度
$O(N^2 \times N^2) = O(N^4)$：O(N²) 种状态，每个状态 4 次转移

### 空间复杂度
$O(N^2 \times N^2) = O(N^4)$：DP 数组大小

### 优点
- ✅ 思路清晰，易于理解
- ✅ 一次遍历解决问题
- ✅ 代码相对简洁

### 缺点
- ❌ 空间占用较大
- ❌ 对于 N=9 会占用约 1.3MB

### 对比其他解法
相比 DFS 记忆化，本解法更直接；相比费用流，本解法更容易实现。
```

### 规范检查清单

- [ ] 四个板块都已包含
- [ ] 每个板块标题为二级标题（##）
- [ ] 问题部分包含完整的问题描述和样例
- [ ] 思路分析清楚阐述了解决思路
- [ ] 代码实现完整可运行
- [ ] 复杂度与优缺点已明确指出
- [ ] 如有多个解法，以三级标题（###）分别展示

## 代码块规范

### 核心原则
- ✅ **保留**：解释性注释和美化性分隔符
  - 变量含义：`int dp[20][10][10]; // dp[步数][人1的行][人2的行]`
  - 初始化说明：`// 初始化为-1表示不可达`
  - 算法步骤：`// 动态规划主循环`
  - 美化分隔符：`// ========== xxx ==========`
  - 关键操作：`// 规范化：保证 a <= b`

- ❌ **删除**：冗余和自解释的注释
  - 避免：`counter++  // 增加计数器`（显而易见，无需注释）
  - 避免：`i = 0;      // 赋值为0`（显而易见，无需注释）

### 语言标记
```markdown
```c       # C/C++ 代码
```python  # Python 代码
```javascript # JavaScript 代码
```go      # Go 代码
```bash    # Bash/Shell 代码
```

### 缩进与空白
- 使用 **4 个空格**作为一个缩进级别
- 相关逻辑块之间用单行空白分隔
- 不过度空白化

正确示例：
```c
for (int k = 2; k < 2 * N; k++) {
    for (int x1 = 1; x1 <= N; x1++) {
        // 处理逻辑
        
        for (int m1 = 0; m1 <= 1; m1++) {
            // 嵌套逻辑
        }
    }
}
```

### 代码长度限制
- 单行不超过 80 个字符（如可能）
- 超过时：优先重组逻辑，次选换行
- 极端情况可超过，但要保持可读性

### 完整代码 vs 片段
- **完整代码**（≤120 行）：包含 `#include` 和 `main()` 等
- **片段代码**（>120 行）：仅展示核心函数，用 `...` 表示省略

## 文本与段落

### 段落规则
- 段落之间用**单行空白**分隔
- 无首行缩进
- 避免过长段落（超过 200 字考虑分段）

### 强调与符号
- **加粗**：关键术语、重要概念
- *斜体*：引用、外文词汇
- `代码`：变量名、函数名、文件名

正确示例：
```markdown
这个**关键概念**用 `variable` 表示。
在 *dynamic programming* 中...
```

## 数学公式

### 行内公式
```markdown
这个公式 $f(x) = x^2$ 表示平方函数。
```

### 块级公式
```markdown
$$
f(x) = \begin{cases}
  x^2 & \text{if } x \geq 0 \\
  -x^2 & \text{if } x < 0
\end{cases}
$$
```

### 配置
- 在 YAML frontmatter 中设置 `math: true`
- 保持 LaTeX 语法简洁

## 列表与表格

### 无序列表
```markdown
- 项目一
- 项目二
  - 子项 2.1
  - 子项 2.2
- 项目三
```

### 有序列表
```markdown
1. 第一项
2. 第二项
   1. 子项 2.1
   2. 子项 2.2
3. 第三项
```

### 表格
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据 | 数据 | 数据 |
```

### 规则
- 列表项前后各留一行空白
- 嵌套最多两层
- 表格行数≤15 行

## 中英文混排

### 间距规则
- **中文与英文间**：空 1 个空格
- **中文与数字间**：空 1 个空格
- **标点符号**：紧贴前文

正确示例：
```
这是一个 Python 例子，版本号是 3.9。

```

### 标点符号
- 中文段落：中文标点 。，；：
- 代码、英文：英文标点 . , ; :

## 常见问题

**Q: 代码块超过 80 字符怎么办？**  
A: 优先重组逻辑，次选换行。极端情况可超过，保持可读性。

**Q: 能否嵌套三层列表？**  
A: 不推荐。改为多个二级列表或重新组织结构。

**Q: 图片放在哪里？**  
A: 相对路径放在文章同目录的 `images/` 文件夹，或用 `/assets/img/` 绝对路径。

**Q: 可以删除注释吗？**  
A: 只删除冗余/自解释的注释。保留所有解释性注释和美化性分隔符。

**Q: 如何处理超长代码？**  
A: 分为多个代码块，或用 `...` 表示省略部分。
